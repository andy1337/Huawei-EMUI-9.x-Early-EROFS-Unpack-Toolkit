# EMUI 9.1 华为早期 EROFS / ext4 / GSI 修补工具

这个目录是独立发布包，只包含稳定脚本，不包含任何镜像、临时分析文件、设备私有文件或其它项目内容。

重要说明：本发布版已经移除“打包回华为老 EROFS”功能。早期 EMUI 9.1 EROFS 的回打压缩布局对内核兼容性和体积非常敏感，当前不作为可发布能力提供。发布版只保留已经稳定验证的解包、ext4 打包、GSI libfs 修补和 raw/sparse 转换。

## 文件清单

- `emui91_erofs_ext4.py`：主入口，负责解包、打包 RW ext4、GSI 修补和 raw/sparse 转换。
- `extract_huawei_erofs.py`：解包华为 EMUI 9.1 早期 EROFS，并生成 `metadata.jsonl`。
- `check_erofs_apks.py`：直接读取华为早期 EROFS 并检查 APK 完整性，支持 raw/sparse 镜像。
- `build_rw_ext4.py`：从解包目录打包 raw RW ext4，保留权限、SELinux xattr、capability xattr。
- `verify_rw_ext4.py`：校验打包后的 ext4 与解包元数据/文件内容一致。
- `extract_ext4_image.py`：从 raw ext4 Android/GSI 镜像导出文件和 `metadata.jsonl`，可在导出时修补 `libfs_mgr.so`。
- `patch_gsi_ext4_vendor.py`：GSI 补丁工具，让 Android 13 arm64 GSI 在挂载 `/vendor` 时使用 ext4。
- `raw_to_android_sparse.py`：raw 镜像转 Android sparse。
- `android_sparse_to_raw.py`：Android sparse 转 raw。
- `ext4_read.py`：读取 raw ext4 文件、目录、inode 元数据和 xattr。
- `ext4_replace_file.py`：在 raw ext4 内原位替换已有文件，GSI 补丁会调用它。

## 环境要求

- Python 3.10 或更新版本。
- 不需要第三方 Python 包。
- Windows PowerShell、Linux shell、macOS shell 都可用；下面示例用 PowerShell。

## 解包华为 EMUI 9.1 EROFS

把 `system.img` 或 `vendor.img` 放到本目录，或在命令里写完整路径。

```powershell
python .\emui91_erofs_ext4.py unpack --partition system system.img system_extracted
python .\emui91_erofs_ext4.py unpack --partition vendor vendor.img vendor_extracted
```

解包完成后会生成：

- 解包目录，例如 `system_extracted` 或 `vendor_extracted`
- `metadata.jsonl`：记录权限、UID/GID、软链接、xattr
- `verify_report.txt`：解包校验报告

确认 `verify_report.txt` 里 `failures=0` 后再继续使用解包目录。

## 从解包目录打包 RW ext4

```powershell
python .\emui91_erofs_ext4.py pack-ext4 --partition system system_extracted system_rw_ext4.img --sparse
python .\emui91_erofs_ext4.py pack-ext4 --partition vendor vendor_extracted vendor_rw_ext4_xattr.img --sparse
```

生成的 sparse 默认名：

- system：`system_rw_ext4_sparse.img`
- vendor：`vendor_rw_ext4_xattr_sparse.img`

刷机一般优先刷 sparse 镜像；如果刷写工具明确要求 raw，再刷 raw。

## 一条命令转换华为 EROFS 到 RW ext4

```powershell
python .\emui91_erofs_ext4.py convert --partition system system.img system_rw_ext4.img --sparse
python .\emui91_erofs_ext4.py convert --partition vendor vendor.img vendor_rw_ext4_xattr.img --sparse
```

如果已经解包过，想复用解包目录：

```powershell
python .\emui91_erofs_ext4.py convert --partition vendor vendor.img vendor_rw_ext4_xattr.img --reuse-extracted vendor_extracted --sparse
```

## GSI ext4 vendor 修补

先 dry-run 检查 GSI 是否支持补丁：

```powershell
python .\emui91_erofs_ext4.py patch-gsi GSI.img --dry-run
```

生成修补后的 raw ext4 GSI：

```powershell
python .\emui91_erofs_ext4.py patch-gsi GSI.img GSI_vendor-ext4-libfs.img
```

这个补丁只修改 GSI 里的 `/system/lib64/libfs_mgr.so`，让 `/vendor` 使用 `ext4` 挂载；不修改 DTS、DTO、kernel、vendor 镜像。

如果输出里能看到 `signature_off`、`arg_off`、`sha256_after`、`readback_ok=True`，说明补丁和回读校验通过。

## 单独处理 raw ext4 GSI

只导出 raw ext4 GSI：

```powershell
python .\emui91_erofs_ext4.py unpack-ext4 GSI.img gsi_extracted --file-data-mode flat --symlink-mode none
```

导出时顺手修补 `libfs_mgr.so`：

```powershell
python .\emui91_erofs_ext4.py unpack-ext4 GSI.img gsi_extracted --file-data-mode flat --symlink-mode none --patch-libfs-vendor-ext4
```

Windows 上建议对 GSI 使用 `--file-data-mode flat`，因为部分 GSI 会同时包含仅大小写不同的文件名。

## 单独校验 ext4

```powershell
python .\emui91_erofs_ext4.py verify --partition vendor vendor_extracted vendor_rw_ext4_xattr.img
```

输出 `failures=0` 才表示打包内容与解包结果一致。

## raw/sparse 转换

```powershell
python .\emui91_erofs_ext4.py sparse vendor_rw_ext4_xattr.img vendor_rw_ext4_xattr_sparse.img
python .\emui91_erofs_ext4.py unsparse vendor_sparse.img vendor_raw.img
```

## 已知范围

- EROFS：稳定支持华为 EMUI 9.1 早期 EROFS system/vendor 解包和 APK 校验。
- ext4：支持从解包目录打包 RW ext4，并保留常见 Android 权限、UID/GID、xattr。
- GSI 补丁：面向 Android 13 arm64、AOSP 风格的 raw ext4 GSI。
- 不提供 EROFS 回打，不提供 DTS/DTO/kernel 补丁。

## 注意事项

- 不要使用普通开源 EROFS 解包器处理这类华为早期 EROFS，APK 等文件可能损坏。
- GSI 输入必须是 raw ext4；如果是 Android sparse，先用 `unsparse` 转 raw。
- 分发给别人时，只发本目录，不要附带镜像或设备私有文件。
- 如果 GSI dry-run 提示 `unsupported libfs_mgr.so`，说明那个 GSI 的 `libfs_mgr.so` 需要单独适配。
